AI Summarization — How It Works

Long articles distilled into bullet points — generated once, cached forever in the DB

1
👤
User opens AI Copilot on a post
Frontend calls POST /posts/:id/summary?language=vi
↓
2
☕
Spring Boot checks DB cache
Summary already exists? Return instantly — no AI call needed
↓
3
🐍
Python AI service receives the article text
POST /summarize · { text, max_bullets: 5, language: "vi" }
↓
4
🤖
Qwen3-4B writes the bullet points
Strips <think> reasoning tokens · returns clean bullet list
↓
5
💾
Summary saved to DB · shown in sidebar
Every reader after this gets it instantly from cache

Summarization — Model Config

Each param tuned to produce factual, concise output — no hallucination, no repetition

temperature
0.2
Low creativity — keeps output factual and grounded, reduces hallucination risk
top_p
0.8
Only considers the top 80% most likely words, cutting off unlikely tokens
repetition_penalty
1.08
Penalizes repeated phrases — stops the model from looping the same sentence
top_k
20
Further narrows each token choice to the top 20 candidates
max_new_tokens
512
Max output length — enough for 5–6 detailed bullet points
model
Qwen3-4B
4B parameter LLM by Alibaba — runs fully on-device, no external API
